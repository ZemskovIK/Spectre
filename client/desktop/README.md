## 🌿 Ветки проекта
- `main` - основная версия с базовой функциональностью и системой аутентификации
- `crypto_protocol` - версия с дополнительным шифрованием данных (в разработке)

## 📋 Описание проекта

GUI-клиент для управления архивом военных писем с системой аутентификации и ролями пользователей. 
Основные возможности:
- 🔐 Авторизация пользователей с разными уровнями доступа
- 📝 CRUD-операции с письмами (для администраторов)
- 👥 Управление пользователями (для администраторов)
- 🔍 Поиск и просмотр писем (для всех авторизованных пользователей)
- 🔑 ECDH обмен ключами для безопасной передачи данных
- 🔒 AES-256 шифрование контента писем

## ✨ Функциональность

### Система аутентификации:
- Вход по логину/паролю
- JWT-токен для сессий
- Ролевая модель (6 уровней доступа)
- Безопасное хранение паролей (bcrypt хеширование)

### Основные возможности:

| Функция                   | Доступность       | Описание                                              |
|---------------------------|-------------------|-------------------------------------------------------|
| Просмотр всех писем       | Все пользователи  | Получение полного списка писем                        |
| Поиск писем               | Все пользователи  | По ID или автору                                      |
| Добавление писем          | Уровень доступа 6 | Создание новых записей                                |
| Редактирование писем      | Уровень доступа 6 | Изменение всех полей письма                           |
| Удаление писем            | Уровень доступа 6 | Удаление по ID с подтверждением                       |
| Просмотр пользователей    | Уровень доступа 6 | Получение списка всех пользователей                   |
| Получение пользователя    | Уровень доступа 6 | Получение информации о пользователе по ID или логину  |
| Создание пользователей    | Уровень доступа 6 | Создание новых пользователей с хешированием           |
| Удаление пользователей    | Уровень доступа 6 | Удаление пользователей по ID                          |

## 🛠️ Установка и запуск

### Требования:
- Python 3.7+
- Зависимости:

```
pip install -r requirements.txt
```

### Запуск:

1. Убедитесь, что сервер API запущен и доступен (порт 5000)
2. Убедитесь, что crypto сервис запущен и доступен (порт 7654)
3. Клонируйте репозиторий
4. Перейдите в директорию desktop
5. Выполните:

```
python main.py
```
> Или запустите main.exe из папки dist

## 🔌 API Endpoints

### Аутентификация:

| Метод	  | Endpoint	      | Действие                   | Примечание           |
|---------|-------------------|----------------------------|----------------------|
| POST	  | /login	          | Аутентификация             | Получение JWT токена |
| GET	  | /ecdh	          | Получение публичного ключа | Для обмена ключами   |
| POST    | /ecdh             | Отправка публичного ключа  | Завершение обмена    |

### Письма (с шифрованием AES-256):

| Метод	  | Endpoint	       | Действие            | Права доступа    |
|---------|--------------------|---------------------|------------------|
| POST	  | /api/letters	   | Создание письма     | Уровень 6        |
| GET	  | /api/letters	   | Чтение всех писем   | Все пользователи |
| GET	  | /api/letters/{id}  | Чтение письма по ID | Все пользователи |
| PUT	  | /api/letters/{id}  | Обновление письма   | Уровень 6        |
| DELETE  | /api/letters/{id}  | Удаление письма     | Уровень 6        |

### Пользователи (админ-панель):

| Метод	  | Endpoint	               | Действие               | Права доступа |
|---------|----------------------------|------------------------|---------------|
| POST	  | /api/users	               | Создание пользователя  | Уровень 6     |
| GET	  | /api/users/{id или логин}  | Получения пользователя | Уровень 6     |
| DELETE  | /api/users/{id}	           | Удаление пользователя  | Уровень 6     |

## 📂 Структура данных

### Письмо (в зашифрованном виде):

```json
{
  "content": ["base64_encoded_encrypted_data"],
  "iv": "base64_iv",
  "hmac": "base64_hmac",
  "nonce": "base64_nonce"
}
```

### Расшифрованное письмо:

```json
{
  "id": 1,
  "author": "Иванов И.И.",
  "body": "Текст письма...",
  "found_at": "1943-05-12T00:00:00Z",
  "found_in": "поле боя",
  "access_level": 1
}
```

### Пользователь:

```json
{
  "id": 1,
  "login": "admin",
  "access_level": 6
}
```

## 🔒 Безопасность

- Обмен ключами: ECDH (Elliptic Curve Diffie-Hellman) на кривой SECP256R1
- Шифрование: AES-256 в режиме CBC с HMAC для целостности
- Пароли: Хеширование bcrypt с солью
- Токены: JWT с подписью HS256
- Уровни доступа: 6 уровней (1 - минимальный, 6 - администратор)
